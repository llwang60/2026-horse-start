## ⚠️ 注意事项

1. **提前测试**：仪式前确保 GitHub Actions 能正常运行
2. **备用方案**：准备 GitHub Web IDE 链接给不熟悉命令行的人
3. **时间控制**：每个环节严格按时间表推进
4. **气氛调动**：适当使用技术梗，但不要让非技术人员感到被排斥
